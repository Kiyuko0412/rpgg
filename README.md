這是我從高二開始，為了挑戰自己並從零開始學習程式開發，自己做的功能完整和能讓多人同時遊玩的機器人。

當初在 Discord 上看到有人做 RPG 機器人，甚至還能營運賺錢，我心裡就想：既然別人能做到，我或許能做得更好！順帶一提，鳩鳩這個名字靈感原自於我非常喜歡的一個遊戲角色。

於是，我從完全不懂 Python 和 SQL 等等的開始，一步步看教學、問 AI、查文件，最終做出了這份作品。



## 表情符號地圖系統

這是我投入最多的部分。因為不希望地圖只是座標或文字，在Discord主要為文字的限制下，我設計了將地圖檔案中的代號轉化成表情符號。

*   地圖沒有寫死，而是在`maps_data.json`可以自由設定：
    *   地圖形狀與佈局
    *   怪物種類與倍率
    *   獎勵：寶箱內容物與結束的結算獎勵，可以是錢、經驗值與道具。
    *   入場費或每日次數限制。
*   在地圖上走動、遇到怪物、寶箱或傳送點，都會更新介面。

## 地圖編輯器

為了讓地圖製作更方便，我找AI用 `tkinter` 寫了GUI工具，做一個關卡之前要手動改十幾分鐘的檔到現在只需要幾分鐘。用滑鼠就能操作，可以直接設計地形、修改地圖資料。

> ![image](https://github.com/Kiyuko0412/rpgg/blob/main/img/2.png)



## 戰鬥與技能系統

核心是一套基於事件的戰鬥系統，雖然目前還非常陽春，但底層架構已經完成且沒有問題。

並且把角色、怪物、技能、效果都寫成類別，讓未來可以更新更多東西。

> ![image](https://github.com/Kiyuko0412/rpgg/blob/main/img/3.png)



## 資料管理與經濟

一個好的 RPG 肯定需要穩定的後台。所以我用QLite來管理整台機器人的運作。

從玩家的等級經驗、背包物品，到每個怪物的詳細設定，全部都由 SQL 來管理。


![image](https://github.com/Kiyuko0412/rpgg/blob/main/img/4.png)
